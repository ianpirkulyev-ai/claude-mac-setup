# Quality vs Speed — какой компромисс выбрать

## Шкала «качество vs скорость»

```
Качество максимум ──────────────────────────────────  Скорость максимум

large-v3        →  large-v3-turbo   →  medium  →  small  →  tiny
эталон             8x быстрее         50-90%     65-75%    50-60% ★
                   1-3% потери        качества   качества  качества

★ — приведено по сравнению с large-v3 на популярных языках. На редких
    языках разница больше.
```

## Матрица решений

### Таблица «задача → модель»

| Задача | Длина аудио | Язык | Рекомендованная модель | Почему |
|---|---|---|---|---|
| TikTok / Reels транскрибат | <1 мин | RU/EN | `large-v3-turbo` | Скорость не критична на коротком, качество максимум |
| Subtitles для shorts (CapCut-стиль) | <1 мин | RU/EN | `large-v3-turbo` + `word_timestamps=True` | Точность слов важнее скорости |
| Подкаст 1-2 ч | 1-2 ч | RU/EN | `large-v3-turbo` | На длинных скорость важна, качество достаточно |
| Подкаст с гостями (diarization) | 1-2 ч | RU/EN | `large-v3-turbo` (whisperx) | + pyannote diarization |
| Интервью на казахском/узбекском | 1-2 ч | KK/UZ | `large-v3` | turbo проседает на редких |
| Конференция / совещание | 2-4 ч | mixed | `large-v3` | Mixed-language, важна точность |
| Translation в EN | любая | любой | `large-v3` | turbo на translation плохо |
| Audiobook | 5-10 ч | один | `large-v3-turbo` или `medium` | Длинно, экономия времени |
| Проверить что записалось | любая | любой | `tiny` | Достаточно «угадывания» |

### Таблица «железо → лучшая модель»

| Железо | RAM/VRAM | Лучшая модель | Реальный realtime factor |
|---|---|---|---|
| RTX 4090 / A100 | 24+ GB | `large-v3` | 10x |
| RTX 4090 / A100 | 24+ GB | `large-v3-turbo` | 17x |
| RTX 3060 / 4060 | 8-12 GB | `large-v3-turbo` | 8-10x |
| RTX 3050 / 4050 | 4-6 GB | `large-v3-turbo` (int8_float16) | 5-7x |
| Mac M4 Pro / Max | 16+ GB | `large-v3-turbo` (MLX) | 18x |
| Mac M2 / M3 Pro | 16 GB | `large-v3-turbo` (MLX) | 15x |
| Mac M1 / M2 base | 8 GB | `medium` или `small` | 8-10x на small, 4x на medium |
| x86 12-16c CPU | 32+ GB | `large-v3-turbo` (int8) | 0.5-0.8x |
| x86 8c CPU | 16 GB | `medium` (int8) | 0.3-0.5x |
| x86 4c CPU | 8 GB | `small` (int8) | 0.2-0.3x |
| Слабый ноут | 4-8 GB | `tiny` или `base` | 0.5-1.5x |

### Когда «достаточно» = `medium`

- Внутренние черновики транскрибации (вычитка не нужна)
- Поиск по аудио (точное распознавание не важно)
- Subtitles для понимания (не для публикации)
- Переводы где LLM пост-обрабатывает текст

### Когда **только** `large-v3` (никаких компромиссов)

- Профессиональные субтитры для публикации (минимум исправлений)
- Юридическая / медицинская транскрибация
- Редкие языки
- Translation task (`task="translate"`)
- Mixed-language audio

## Бенчмарки WER (Word Error Rate)

WER = доля неправильных слов. Чем меньше тем лучше.

### Английский (LibriSpeech test-clean)

| Модель | WER | Realtime (RTX 4090) |
|---|---|---|
| `tiny` | 12.0% | 80x |
| `base` | 9.0% | 60x |
| `small` | 6.0% | 40x |
| `medium` | 4.0% | 20x |
| `large-v3` | **2.0%** | 10x |
| `large-v3-turbo` | 2.5% | 17x |
| `distil-large-v3` | 2.0% | 25x (EN-only) |

### Русский (Common Voice 17, ru)

| Модель | WER |
|---|---|
| `tiny` | 35% |
| `base` | 28% |
| `small` | 18% |
| `medium` | 12% |
| `large-v3` | **9%** |
| `large-v3-turbo` | 11% |

### Казахский (Common Voice 17, kk)

| Модель | WER |
|---|---|
| `small` | 30% |
| `medium` | 22% |
| `large-v3` | **15%** |
| `large-v3-turbo` | 22% |

> ⚠️ На казахском `turbo` теряет **7%** против `large-v3` — это много. Для KK/UZ/TT/AR редких языков **строго `large-v3`**.

## Что улучшает качество (ortogonal к выбору модели)

### Препроцессинг аудио

1. **Нормализация громкости**:
   ```bash
   ffmpeg -i input.mp3 -af loudnorm output.wav
   ```
2. **Denoise** на шумных записях:
   ```bash
   pip install noisereduce
   # или используй RNNoise (https://github.com/xiph/rnnoise)
   ```
3. **Удаление музыки** (если есть фон):
   ```bash
   pip install demucs
   demucs --two-stems=vocals input.mp3
   ```

### Initial prompt

Подсказывай контекст модели — она лучше угадывает редкие слова:

```python
result = model.transcribe(
    audio,
    language="ru",
    initial_prompt=(
        "Это интервью с разработчиком. Спикер использует термины: "
        "API, фреймворк, библиотека, deployment, latency, throughput, "
        "Python, JavaScript, TypeScript, Docker, Kubernetes."
    ),
)
```

Особенно помогает на:
- Технических подкастах (термины)
- Медицинских записях (диагнозы / препараты)
- Юридических (термины кодексов)
- Бренд-неймы (имена компаний / продуктов)

### Post-processing

Whisper иногда косячит с:
- Пунктуацией → прогон через LLM с промптом «расставь пунктуацию»
- Капитализацией → то же самое
- Названиями (`айфон` вместо `iPhone`) → LLM-чек по словарю

См. [methodology/post-processing.md](post-processing.md) для готовых промптов.

## Когда переключаться

### От большой модели к меньшей

✅ Транскрибация — внутренняя черновая (для поиска / индексации)
✅ Объёмы критичны (10+ часов в день)
✅ Стоимость времени важнее качества (нужно успеть до дедлайна)

### От меньшей к большой

⚠️ Видишь повторяющиеся ошибки в одних и тех же словах
⚠️ Спикер с акцентом / нестандартным голосом
⚠️ Низкое качество исходника (диктофон, телефон)
⚠️ Перевод (`task="translate"`)
⚠️ Редкий язык / диалект

## Финальная шпаргалка

**По дефолту** для всех задач: `large-v3-turbo` + явный `language` + `vad_filter=True`.

**Если** что-то «звучит не так»:
1. Сначала попробуй `large-v3` (то же железо, чуть медленнее, +1-2% качества)
2. Потом `initial_prompt` с контекстом
3. Потом препроцессинг (denoise, normalize, demucs)
4. Только в самом конце — fine-tuning или другая модель

**Если** хочешь чек-лист пунктом за пунктом до запроса в production:
1. Аудио в 16kHz mono WAV (`ffmpeg -ar 16000 -ac 1`)
2. Включён VAD (`vad_filter=True`)
3. Указан язык (`language="ru"`)
4. Compute type правильный для железа (`float16` GPU / `int8` CPU)
5. word_timestamps только если нужны
6. На длинных файлах — batched mode
7. Диариация если 2+ спикеров (отдельный шаг whisperx)

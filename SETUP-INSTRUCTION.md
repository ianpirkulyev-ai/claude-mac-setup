# Инструкция для Claude Code: настрой этот MacBook с нуля

Привет, Claude. Ты запущен в Claude Code на новом MacBook. Перед тобой человек,
который только начинает (это не разработчик, относись бережно). Твоя задача —
**настроить ему рабочую среду с нуля**: базовые инструменты, методологию работы с тобой
(«смысло-кодинг») и локальную диктовку/транскрибацию голоса (Whisper).

Действуй пошагово. После каждого крупного шага коротко докладывай человеку простым
языком, что сделал. Если шаг требует пароля/подтверждения или открытия настроек системы —
останови и объясни, что и зачем нажать.

> ⚠️ Это чистый старт «под себя». Здесь НЕТ чужих проектов и чужих данных — человек
> наполнит всё сам. Ничего приватного устанавливать не нужно.

## Что у тебя есть рядом с этой инструкцией

Этот файл лежит в репозитории `claude-mac-setup`. Рядом, в той же папке, есть каталог

человека склонировать его полностью (`git clone`).

## Шаг 0. Сориентируйся

Выполни и покажи человеку, что обнаружил (чип Apple Silicon или Intel — от этого зависит шаг 5):

```bash
sw_vers && uname -m && echo "---" && which brew git python3 node claude 2>/dev/null
```

`arm64` = Apple Silicon (M1/M2/M3/M4) — основной путь. `x86_64` = Intel Mac.

---

## Шаг 1. Базовые инструменты (Homebrew + git + node + python)

1. **Homebrew** (если `which brew` ничего не вернул):

   ```bash
   /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
   ```

   После установки на Apple Silicon добавь brew в PATH (инсталлятор подскажет точные строки,
   обычно это `eval "$(/opt/homebrew/bin/brew shellenv)"` + дописать в `~/.zprofile`).

2. **Инструменты:**

   ```bash
   brew install git gh ffmpeg python@3.12 node
   ```

   - `git` — версионирование, `gh` — GitHub CLI, `ffmpeg` — для аудио/видео (нужен Whisper),
     `python@3.12` — для Whisper (НЕ 3.14 — под неё ещё нет нужных пакетов), `node` — для Claude Code.

3. Проверь:

   ```bash
   git --version && python3.12 --version && node --version && ffmpeg -version | head -1
   ```

## Шаг 3. Методология работы («смысло-кодинг»): стартер проектов

Это главное, что делает работу с тобой осмысленной. Есть публичный шаблон —
человек будет разворачивать его при старте **каждого нового проекта**.

1. Покажи человеку, что это и зачем (своими словами): шаблон даёт структуру проекта —
   бизнес-контекст (`.business/`), планы (`plans/`), рефлексию (`retrospectives/`),
   готовые промпты (`prompts/`) и пошаговый онбординг (`AUTOPILOT.md`). Не код вперёд,
   а сначала смысл: для кого, зачем, что делаем.

2. Источник (канонический, публичный):
   **https://github.com/artemiimillier/claude-code-starter**

3. Прямо сейчас разворачивать проект НЕ надо (проектов ещё нет). Просто запомни и
   объясни человеку правило: **когда он захочет начать новый проект**, ты сначала
   склонируешь шаблон и развернёшь структуру:

   ```bash
   # делается в момент старта конкретного проекта, не сейчас:
   git clone https://github.com/artemiimillier/claude-code-starter ~/Documents/<имя-проекта>
   # дальше — по файлу USAGE.md внутри шаблона
   ```

4. (Опционально, если человек хочет.) Сделать локальный «вендор» шаблона, чтобы потом
   разворачивать без интернета:

   ```bash
   git clone https://github.com/artemiimillier/claude-code-starter ~/project-starter
   ```

## Шаг 4. «Второй мозг» в Obsidian (паттерн LLM Wiki Андрея Карпаты)

Это персональная база знаний в Markdown, которую **ведёт AI-агент** (ты, Claude). Идея
Андрея Карпаты («LLM Wiki»): не заставлять модель каждый раз выводить всё с нуля, а держать
постоянную связанную базу заметок, которую агент сам наполняет и поддерживает. Хранилище —
Obsidian (это просто папка с `.md`-файлами, ничего не уходит в облако).

> Каркас ниже **пустой** — это только структура. Человек наполняет её сам. Ядро
> (`index.md` / `agents.md` / `log.md` + папки `raw/`, `wiki/`, `journal/`, `crm/`) — из
> оригинального паттерна Карпаты. Имена вроде `Inbox / Evergreen / MOCs` — устоявшееся
> соглашение сообщества «в стиле Карпаты», можно переименовать под себя.

### 4.1. Установи Obsidian

```bash
brew install --cask obsidian
```

(или скачать с https://obsidian.md и перетащить в «Программы».)

### 4.2. Разверни пустой каркас базы знаний

```bash
VAULT=~/Documents/second-brain
mkdir -p "$VAULT"/{raw/processed,wiki,journal,crm,Inbox,Evergreen,MOCs,Projects,Archive,Attachments}

cat > "$VAULT/index.md" <<'EOF'
# Index — карта базы знаний

Корневая точка входа. Здесь оглавление и ссылки на ключевые разделы (MOCs).

- [[agents]] — как AI-агент ведёт эту базу
- wiki/ — статьи (устоявшееся знание)
- journal/ — дневник по датам
- raw/ — сырые входящие материалы
- crm/ — люди и контакты
EOF

cat > "$VAULT/agents.md" <<'EOF'
# Agents — правила ведения базы для AI

Инструкция для Claude, который поддерживает эту базу знаний.

## Что делает агент
- Принимает сырьё из `raw/`, осмысляет и раскладывает в `wiki/`, помечает обработанное в `raw/processed/`.
- Пишет и обновляет статьи в `wiki/` — связно, со ссылками `[[...]]` между заметками.
- Ведёт дневник в `journal/` (файлы вида `YYYY-MM-DD.md`).
- Каждое заметное изменение фиксирует строкой в `log.md`.

## Принципы
- Не выдумывать факты. Нет данных — помечать «гипотеза».
- Коротко и по-человечески, без воды.
- Всё локально, наружу ничего не отправлять.
EOF

cat > "$VAULT/log.md" <<'EOF'
# Log — журнал изменений базы

<!-- агент дописывает строки сверху: YYYY-MM-DD — что изменилось -->
EOF

echo "Каркас создан в $VAULT"
ls -1 "$VAULT"
```

### 4.3. Открой хранилище в Obsidian

Запусти Obsidian → **Open folder as vault** → выбери `~/Documents/second-brain`.
Объясни человеку: дальше он просто кидает мысли/файлы в `raw/` и `journal/`, а тебя
(Claude) просит «разложи и обнови вики» — ты ведёшь базу по правилам из `agents.md`.

## Шаг 5. Whisper — диктовка голосом + транскрибация файлов

Локально, бесплатно, без OpenAI API. Заменяет Superwhisper / Wispr Flow ($8–12/мес).

### 5.1. Установи скилл

```bash
mkdir -p ~/.claude/skills
# скопируй каталог whisper-skill из этого репозитория в ~/.claude/skills/
# (запускай из корня склонированного claude-mac-setup):
cp -R whisper-skill ~/.claude/skills/
ls ~/.claude/skills/whisper-skill
```

### 5.2. Поставь Whisper-бэкенд

**Apple Silicon (arm64)** — нативный Metal, быстрый:

```bash
mkdir -p ~/whisper-projects && cd ~/whisper-projects
python3.12 -m venv .venv && source .venv/bin/activate
pip install --upgrade pip
pip install mlx-whisper yt-dlp
# для диктовки (push-to-talk демон):
pip install sounddevice soundfile pynput pyperclip pystray Pillow numpy
mlx_whisper --help    # проверка
```

**Intel Mac (x86_64)** — MLX не работает, ставь faster-whisper (медленнее):

```bash
mkdir -p ~/whisper-projects && cd ~/whisper-projects
python3.12 -m venv .venv && source .venv/bin/activate
pip install --upgrade pip
pip install faster-whisper yt-dlp
pip install sounddevice soundfile pynput pyperclip pystray Pillow numpy
```

> Альтернатива для не-технаря: внутри `~/.claude/skills/whisper-skill/` есть
> `python wizard.py` — интерактивный мастер, который сам определит железо и всё поставит.
> Можешь предложить его человеку вместо ручных команд выше.

### 5.3. Транскрибация файла (проверка, что работает)

```bash
cd ~/whisper-projects && source .venv/bin/activate
# любой mp3/mp4/m4a; модель скачается автоматом (~1.6 ГБ, один раз):
mlx_whisper --model mlx-community/whisper-large-v3-turbo --language ru \
            --output-dir ./out путь/к/файлу.mp3
```

(На Intel — используй примеры из скилла: `python -m examples.transcribe_one файл.mp3`,
запускать из папки `~/.claude/skills/whisper-skill/` в активированном venv.)

### 5.4. Диктовка голосом (push-to-talk)

```bash
cd ~/.claude/skills/whisper-skill
source ~/whisper-projects/.venv/bin/activate
python -m examples.voice_dictation --setup    # одноразово создаёт конфиг
python -m examples.voice_dictation            # запуск, висит в трее
```

Как пользоваться: зажать `Ctrl+Shift+Space` → говорить → отпустить → текст вставится
в активное поле (Slack, заметки, браузер — куда угодно). Язык по умолчанию русский,
меняется в `~/.config/whisper-skill/voice_dictation.json`.

> ⚠️ **macOS попросит доступ** к Микрофону и к «Универсальному доступу / Accessibility»
> (чтобы вставлять текст и слушать хоткей). Останови человека и проведи его:
> Системные настройки → Конфиденциальность и безопасность → Микрофон / Универсальный доступ →
> включить Терминал (или приложение, из которого запущено). Без этих разрешений диктовка
> промолчит.
